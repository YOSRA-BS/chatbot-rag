WHAT IS GENERATIVE AI?
-----------------------
Generative AI is a branch of artificial intelligence that focuses on creating
new content such as text, images, code, music, and video using models like
transformers, GANs, and diffusion models.

It is used in tools like ChatGPT, Claude, Gemini, and DALL-E and is widely
applied in automation, chatbots, and personalization. This tutorial offers a
step-by-step guide to all major concepts and techniques required to learn and
build GenAI applications, with practical projects and modern frameworks.

Generative AI is a type of artificial intelligence designed to create new content
such as text, images, music, or code by learning patterns from existing data.
These models generate original outputs that are often indistinguishable from
human-created content. They use techniques like deep learning and neural networks.

Unlike discriminative AI which focuses on classifying data into categories
(like spam vs. not spam), generative AI creates new data such as text, images,
audio, or video that resembles real-world examples.

SECTION 1 - AI AND DEEP LEARNING FOUNDATIONS
---------------------------------------------
Understanding the foundations of AI and deep learning is essential for working
with GenAI models.

Key topics:
- What is Artificial Intelligence?
- Neural Networks: Layers of nodes inspired by the human brain
- RNNs (Recurrent Neural Networks): Process sequences of data, predecessor to Transformers
- LSTMs (Long Short-Term Memory): A type of RNN that handles long-term dependencies
- Transformers and Self-Attention: The architecture powering all modern LLMs
- Autoencoders and Latent Space: Compress data into a latent representation for generation
- GANs (Generative Adversarial Networks): Two networks competing to generate realistic data
- Diffusion Models: Generate images by learning to reverse a noise-adding process

HOW GENERATIVE AI GENERATES CONTENT
--------------------------------------
Generative AI is trained on large datasets (text, images, audio, or video) using
deep learning networks. During training, the model learns parameters (millions or
billions of them) that help predict or generate content.

Models generate output based on learned patterns and prompts:
- Text: Uses LLMs to predict the next token in a sequence, enabling coherent
  paragraph or essay generation
- Images: Uses diffusion models (Stable Diffusion, DALL-E) or GANs to generate
  visually coherent images from text prompts
- Code: Uses code-specialized LLMs (CodeLlama, GitHub Copilot) to generate
  accurate code based on natural language instructions
- Audio: Uses audio generation models to create music, speech, and sound effects
- Video: Uses multimodal systems like Sora by OpenAI or Runway to generate short,
  temporally coherent video clips from text prompts

SECTION 3 - PROMPT ENGINEERING
--------------------------------
Prompt engineering is the practice of crafting inputs to get better outputs from LLMs.

Key prompt engineering techniques:
1. Zero-Shot Prompting:
   Asking the model to perform a task without any examples.
   Example: "Classify this review as positive or negative: [review text]"

2. One-Shot Prompting:
   Providing one example before asking the model to perform a task.

3. Few-Shot Prompting:
   Providing 2-5 examples before asking the model.
   This significantly improves performance on classification and formatting tasks.

4. Chain of Thought (CoT) Prompting:
   Asking the model to think step by step before giving an answer.
   Dramatically improves performance on math, logic, and multi-step reasoning.
   Triggered by phrases like "Think step by step" or "Let's work through this."

5. Role and Contextual Prompting:
   Giving the model a specific identity: "You are a senior software engineer..."
   Setting context makes the model respond in the appropriate style and domain.

6. ReAct (Reasoning + Acting) Prompting:
   Combines reasoning with tool use. The model reasons about what action to take,
   acts using a tool (web search, calculator), and observes the result.

7. Retrieval-Augmented Prompting (RAG):
   Retrieving relevant documents and including them in the prompt as context.
   Reduces hallucination and enables answering questions about private data.

8. Self-Consistency Prompting:
   Generating multiple reasoning chains and picking the most consistent answer.

SECTION 7 - POPULAR GENERATIVE AI MODELS
-----------------------------------------

Autoregressive Transformer Models:
Generate sequences by predicting the next token based on all previous ones,
moving step by step through the text. The architecture relies on the transformer
self-attention mechanism to capture context from the entire input.
Popular examples: GPT-4, Claude, LLaMA, Mistral.

Diffusion Models:
Generate data such as images or audio by starting with pure random noise and
gradually refining it into a coherent output through a series of denoising steps.
Each step reverses a simulated diffusion process that added noise to real data
during training. Popular examples: Stable Diffusion, DALL-E 3, Midjourney.

GANs (Generative Adversarial Networks):
Use two competing networks: a generator (creates fake data) and a discriminator
(distinguishes real from fake). Training continues until the generator fools the
discriminator. Largely superseded by diffusion models for image generation.

Variational Autoencoders (VAEs):
A variation of the Autoencoder that adds a probabilistic approach to the encoding
process. Used for generating new data similar to the input and commonly applied
in image generation and other creative tasks.

FINE-TUNING IN GENERATIVE AI
-------------------------------
Generative AI models can be further trained on domain-specific labeled data to
adapt to specialized tasks or stylistic needs. This technique is called fine-tuning.

Modern fine-tuning methods:
- Full Fine-Tuning: Updates all model weights, very expensive
- LoRA: Low-rank adaptation, updates only small matrices added to frozen weights
- QLoRA: Quantized LoRA, enables fine-tuning on consumer GPUs
- RLHF: Uses human feedback to align model outputs with human preferences
- DPO: Direct Preference Optimization, simpler alternative to RLHF
